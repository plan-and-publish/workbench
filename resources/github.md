# Overview

This document includes info about github repositories, workflows and relations, their address and a short description of what is in them. This can be used by agents to create and track issues, and anything related to finding the most relevant repository.

## Projects

- The main project that all the epics and parent issues are listed in is [Team Backlog](https://github.com/orgs/plan-and-publish/projects/2)
- Epics should always be created and provided by user
- Agents can search the epics to find the best relevant epic to create tasks and other sub-issues under.
- Sub-issues that are created under the epics in this project should always be created in their relevant repository.
